

Projeto de Simulação de Serviços de Redes de Cinemas

Esta é uma aplicação back-end que simula diversos serviços relacionados a redes de cinemas, oferecendo funcionalidades como:

Verificar quais cinemas existem em uma determinada cidade.
Consultar quais filmes estão em cartaz, incluindo os horários das sessões e quais cinemas e salas estão exibindo cada filme.
Identificar quais filmes são estreias.
Consultar o preço dos ingressos.
Cadastrar, atualizar e/ou remover cidades, cinemas, salas, sessões e filmes.
Conhecimentos Utilizados Neste Projeto
Desenvolvimento de APIs e API Gateway.
Banco de dados MongoDB: Implementação de operações CRUD, blacklist e TTL de tokens, além de controle de níveis de usuário.
Registro de logs: Utilização da biblioteca Winston.
Validação de dados: Verificação de dados obrigatórios e seus respectivos formatos e regras com a biblioteca Joi.
Criação e validação de tokens: Uso da biblioteca jsonwebtoken.
Controle de acesso: Implementação de método de autorização para determinadas rotas.
Testes automatizados: Realização de testes para todas as funções/métodos usando a biblioteca Jest.

Como Usar:

Sinta-se à vontade para explorar ou utilizar o que foi abordado neste projeto. Para executar a aplicação, siga os passos abaixo:

Instalação das Dependências: Execute o comando a seguir no terminal, dentro do repositório do projeto: npm install.
Em seguida, inicie a aplicação com: "npm start" ou, para o ambiente de desenvolvimento, "npm run dev".

Configuração do Banco de Dados:

Se você deseja utilizar um banco já configurado com dados inseridos, extraia os arquivos do "data.zip", que contém a pasta "data" para cada um dos serviços: cinema-catalog-services, api-gateway e movies-services. Lembre-se de renomear as pastas para que fiquem apenas como "data". Por exemplo, renomeie "data movies-services" para "data".

Caso prefira criar seus próprios bancos de dados, utilize os dados de modelo disponíveis no repositório "seeds" de cada microserviço.

Instâncias de Banco de Dados:

É importante lembrar que cada microserviço deve ter sua própria instância de banco de dados. No meu caso, configurei uma instância para cada microserviço, utilizando as portas 27017, 27018 e 27019. No entanto, você pode optar por criar um único banco de dados para todos os microserviços. Para isso, crie um arquivo .env e defina os valores desejados para as variáveis de ambiente. Consulte o arquivo .env.example para saber quais variáveis são necessárias.
